🍪 JavaScript Cookies

📅 Last Updated : 11 Jun 2026
A cookie is a small piece of information that persists between a server and a client. Web browsers store this information at the time of browsing. Cookies are typically stored as name-value pairs separated by semicolons.

They help maintain user state and remember user information across all web pages, enabling features like session management, personalization, and tracking.

⚙️ How Cookies Work

Cookies enable the server to recognize returning users. Here's the step-by-step flow:

1
User sends request

Browser sends a request to the server. Each request is treated as a new user unless a cookie identifies them.

2
Server sets cookie

The server adds a Set-Cookie header in the response. The browser stores it locally.

3
Cookie sent automatically

On subsequent requests, the browser automatically includes the cookie in the request header, allowing the server to recognize the user.

📌 Key Point: In JavaScript, cookies are accessed and manipulated via the document.cookie property. You can create, read, update, and delete cookies using this interface.

📝 Example 1: Setting & Retrieving a Cookie

This example demonstrates how to create a cookie using JavaScript and retrieve its value. Click the buttons below to set and display the cookie.

Basic Set & Get Cookie

No cookie set yet.
                function setBasicCookie() {
                document.cookie = "username=Duke Martin";
                }

                function getBasicCookie() {
                if (document.cookie.length != 0) {
                alert(document.cookie);
                } else {
                alert("Cookie not available");
                }
                }

                function deleteBasicCookie() {
                document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 UTC; path=/;";
                location.reload();
                }
✅ Result: When you click "Set Cookie", a cookie named username with value Duke Martin is stored. Click "Get Cookie" to display the full cookie string.

🔍 Example 2: Displaying Name & Value Separately

This example shows how to split the cookie string into its name and value parts, displaying them individually.

Parsing Name & Value Split

No cookie set yet.
                function setSplitCookie() {
                document.cookie = "username=Duke Martin";
                }

                function getSplitCookie() {
                if (document.cookie.length != 0) {
                var array = document.cookie.split("=");
                alert("Name=" + array[0] + " " + "Value=" + array[1]);
                } else {
                alert("Cookie not available");
                }
                }

                function deleteSplitCookie() {
                document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 UTC; path=/;";
                location.reload();
                }
💡 Tip: Splitting the cookie string with split("=") gives you array[0] as the name and array[1] as the value. This is useful when you need to work with the cookie data programmatically.

🎨 Example 3: Storing User Preferences (Background Color)

This example demonstrates how cookies can remember a user's preference — in this case, a selected background color. The choice is stored in a cookie and automatically applied when the page reloads.

Preference Remember Background Color

(preview)
No color preference stored.
                function applyColorPreference() {
                var select = document.getElementById("colorSelect");
                var value = select.value;
                if (value != "Select Color") {
                document.bgColor = value;
                // Also update the swatch
                document.getElementById("colorSwatch").style.backgroundColor = value;
                document.cookie = "color=" + value;
                }
                }

                window.onload = function() {
                if (document.cookie.length != 0) {
                var array = document.cookie.split("=");
                if (array[0] === "color") {
                var color = array[1];
                document.bgColor = color;
                document.getElementById("colorSelect").value = color;
                document.getElementById("colorSwatch").style.backgroundColor = color;
                }
                }
                }

                function clearColorPreference() {
                document.cookie = "color=; expires=Thu, 01 Jan 1970 00:00:00 UTC; path=/;";
                document.bgColor = "#ffffff";
                document.getElementById("colorSelect").value = "Select Color";
                document.getElementById("colorSwatch").style.backgroundColor = "#ffffff";
                location.reload();
                }
🎯 How it works:
  1. Select a color from the dropdown and click "Apply & Save Color".
  2. The color is saved in a cookie named color.
  3. When you reload the page, the window.onload event reads the cookie and automatically restores your preferred background color.
⚠️ Note: The background color change is applied to the entire page (including the container). This demo uses document.bgColor for simplicity. In modern applications, you'd typically use CSS variables or class-based theming.

📋 Cookie Attributes – Quick Reference

Attribute Description Example
expires Cookie valid until a specific date & time expires=Sun, 20 Aug 2030 12:00:00 UTC
max-age Cookie valid for a duration in seconds max-age=31536000
path Defines the URL path where cookie is accessible path=/
domain Specifies the domain (and subdomains) for the cookie domain=example.com
📚 Learn more: For a deeper dive into cookie attributes like expires, max-age, path, and domain, check out the JavaScript Cookie Attributes guide.
✅ Conclusion: JavaScript cookies are a fundamental tool for maintaining state, remembering user preferences, and enabling personalized web experiences.

Best practices:
  • Keep cookies small (under 4KB).
  • Never store sensitive information (passwords, credit cards) in cookies.
  • Use Secure and HttpOnly flags for enhanced security.
  • Always set an expiration (or use max-age) for persistent cookies.
  • Consider using path=/ to make cookies available site-wide.
